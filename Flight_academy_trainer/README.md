# Flight Academy Trainer — вимоги до розділу

Розділ «слухай + читай синхронно» (тег **Listen & Read** у меню): програвач MP3 +
транскрипт, у якому активна відповідь підсвічується в такт аудіо (автоскрол,
клік-на-відповідь → перемотка, контроль швидкості). Усе self-contained, без бандлера.

Кожен **курс** — це окрема standalone HTML-сторінка зі своїм рушієм усередині та
своєю папкою аудіо. Зараз тут два курси:

- `flight_academy_trainer.html` — Pilot Guide LSA, 7 розділів, аудіо в `Pilot_Guide_LSA/`.
- `belgrade_takeoff_accident_trainer.html` — Embraer E190 Belgrade, аудіо в `Belgrade_Accident/`.

Новий курс **завжди** = нова папка аудіо + нова HTML-сторінка + нова картка в `index.html`.
Не змішувати курси в одному файлі.

## Як працює рушій (контракт даних)

Сторінку керує один JS-масив `CHAPTERS` і константа шляху `DIR`:

```js
const DIR = 'НАЗВА_ПАПКИ/';            // папка з MP3 цього курсу
const CHAPTERS = [
  { num: 1, title: "Назва розділу", mp3: "01-lowercase-hyphen.mp3", segments: [
    { q: "Заголовок/питання (показується зверху)", a: "Дослівний текст, що звучить і підсвічується." },
    ...
  ]},
  ...
];
```

- `audio.src = DIR + ch.mp3` — тобто **один розділ = один MP3-файл** з папки `DIR`.
- `q` — короткий заголовок секції (рендериться як `.seg-q`, в синхронізації **не бере участі**).
- `a` — дослівний озвучений текст (рендериться як `.seg-a`, саме його підсвічує плеєр).
- Синхронізація MVP: тривалість MP3 ділиться між сегментами **пропорційно довжині `a`**
  (`s.a.length`). Точного потаймкодового вирівнювання поки немає.

## Вимоги до файлів: що, куди і після якої обробки

**Класти в репозиторій (комітити):**

- Папка курсу `Flight_academy_trainer/<course-folder>/` з **готовими MP3** розділів,
  імена `NN-lowercase-hyphen.mp3` (порядковий номер + slug), наприклад `03-normal-flight-procedures.mp3`.
- HTML-сторінка курсу `Flight_academy_trainer/<course>_trainer.html`.
- Транскрипт-джерело `<course>.md` — **опційно**, як provenance. Рушій його не читає
  (текст вшито в `CHAPTERS`), але зручно тримати поряд для майбутніх правок/вирівнювання.

**НЕ класти в репозиторій (робочі артефакти):**

- Проєкт Audacity: `*.aup3`, `*.aup3-shm`, `*.aup3-wal` — великі (сотні МБ), бінарні,
  не потрібні сайту. Зберігай локально / в особистому бекапі, **не комітити**.
- `.DS_Store`, тимчасові експорти, проміжні WAV.
- Додай шаблон папки-джерела у `.gitignore`, якщо тримаєш сирці поряд.

Тобто: Audacity-проєкт **зберігати — так, але поза git**. У репозиторій іде лише
**експортований результат** (MP3) + HTML + (опційно) `.md`.

## Розбивати MP3 на розділи? — Так (рекомендація)

Два варіанти структури:

- **Per-chapter (рекомендовано).** Один MP3 на кожен логічний розділ → повноцінна
  «рейка» розділів, точніша синхронізація (похибка довжини обмежена коротким розділом),
  зручна навігація. Так зроблено Pilot Guide LSA (7 файлів = 7 розділів). Saved-up
  нарізки на секції (як 14 файлів Белграда) — це фактично готові розділи.
- **Single-master (швидкий шлях).** Один склеєний MP3 на весь курс, поділ на сегменти
  лише за довжиною тексту. Простіше, але рейка з однієї кнопки і синхрон «пливе» на
  довгому аудіо. Прийнятно тільки для короткої лекції.

**Правило:** якщо матеріал має ≥ 3 логічні секції — розбивати на per-chapter MP3.
Один файл на розділ; назва `NN-<slug>.mp3` за порядком звучання.

## Покроковий пайплайн додавання курсу

1. **Контент.** Підготувати `<course>.md` — дослівний текст лекції, розбитий на секції.
2. **Аудіо.** Озвучити/змонтувати в Audacity. Експортувати **по одному MP3 на розділ**,
   назви `NN-lowercase-hyphen.mp3` у порядку звучання. Audacity-проєкт лишити локально.
3. **Папка.** Створити `Flight_academy_trainer/<course-folder>/`, покласти туди MP3
   (і, за бажання, `<course>.md`).
4. **Сторінка.** Скопіювати наявний `*_trainer.html` як `<course>_trainer.html` і змінити:
   - `<title>`;
   - `const DIR = '<course-folder>/';`;
   - масив `const CHAPTERS = [...]` (свої розділи: `num`, `title`, `mp3`, `segments`);
   - hero-блок: `.kicker`, `<h1>`, вступний `<p>`;
   - дефолтний текст `#nowPlaying` (перший розділ).
   - **Не чіпати** решту рушія; back-link `← All quizzes` (зверху + у футері) → `../index.html`
     має лишитися.
5. **Меню.** Додати запис у масив `MODULES` у `index.html`:
   `{ cat: "interactive", tag: "Listen & Read", title: "...", desc: "...",
   href: "Flight_academy_trainer/<course>_trainer.html", cta: "Open Trainer" }`.
6. **Перевірка.** Локально (`python3 -m http.server`): сторінка 200, MP3-и 200,
   рейка показує всі розділи, підсвітка йде в такт, клік по сегменту перемотує.
7. **Публікація.** `git add` тільки MP3 + HTML (+ `.md`) і `index.html`; `git push`;
   звірити live-URL.

## Іменування

- Папки нових курсів і MP3 — `lowercase-hyphen` (узгоджено з рештою сайту).
  Наявні `Pilot_Guide_LSA/` лишаються як є; нові робити в нижньому регістрі через дефіс.
- MP3 розділу: `NN-<slug>.mp3` (двоцифровий порядок + назва розділу).

## Точна синхронізація (на майбутнє)

Підпис на сторінці чесно каже, що sync приблизний. Коли потрібна точність —
два шляхи:

- **З нарізок:** якщо є окремі MP3 на секцію, реальні тривалості дають точні межі
  без додаткових інструментів (sum = майстер).
- **Forced alignment:** `aeneas` по парі (аудіо, текст) → `chapters.json` з `start/end`
  на речення; рушій тоді читає таймкоди замість оцінки за довжиною.
