# ТЗ: Усний тренажер «Active Recall & Audio Drill»

## 1. Призначення та концепція

**Active-Recall / Spoken-Fluency тренажер** для автоматизації стійких словосполучень (chunks / collocations).

Замість механічного перекладу або зазубрювання ізольованих слів, користувач отримує контекстуальне визначення або комунікативну задачу англійською, самостійно формулює і **вимовляє цільову фразу вголос**, а потім миттєво перевіряє себе за допомогою тексту-еталону та інтегрованого аудіо-супроводу (TTS).

## 2. Архітектура даних (Data Schema)

Дані зберігаються у локальних JSON-файлах уроків. Тренажер активується лише за наявності масиву `drills` у структурі даних.

### Формат JSON (Поля об'єкта)

Кожен елемент масиву `drills` — це структурований об'єкт із трьома полями:

- `task` — дескрипція/визначення ситуації англійською мовою (без використання цільового слова).
- `target` — еталонна стійка фраза (collocation), яку потрібно згадати й вимовити.
- `audio_id` *(опціонально)* — ідентифікатор попередньо згенерованого TTS-файлу для озвучення еталону.

JSON

```
"drills": [
  {
    "task": "Use the professional medical phrase for performing chest compressions and rescue breaths.",
    "target": "administer CPR",
    "audio_id": "drill_01_cpr"
  },
  {
    "task": "Say that the flight continued its journey again after a temporary interruption or unscheduled stop.",
    "target": "flight resumed",
    "audio_id": "drill_01_resumed"
  }
]
```

## 3. Функціональні вимоги та UI/UX Логіка

### Стан додатка (State Management)

- `items`: масив об'єктів з JSON.
- `history`: масив індексів уже пройдених карток (для уникнення повторів).
- `currentIndex`: індекс поточної активної картки (старт: `-1`).
- `isRevealed`: булеве значення, що контролює видимість еталону (`false` за замовчуванням).

### Життєвий цикл та логіка взаємодії (Покроково)

1. **Стартовий стан:**
   - Поле картки показує інструкцію: *“Read the description, say the collocation out loud, then check yourself.”*
   - Кнопка **«Next Drill»** активна.
   - Кнопка **«Show Answer»** та кнопка аудіо-плеєра — `disabled`.
2. **Дія: Клік на «Next Drill»**
   - Генерується випадковий індекс, який **не дорівнює** поточному і якого **немає** в `history` (якщо довжина `history` < `items.length`). Якщо всі картки пройдені, `history` обнуляється.
   - Текст завдання (`task`) виводиться на екран.
   - Поле відповіді (`target`) повністю приховане.
   - `isRevealed` встановлюється в `false`.
   - Кнопка **«Show Answer»** стає активною.
3. **Дія: Клік на «Show Answer»**
   - Поле відповіді плавно відображається на екрані.
   - Кнопка **«Show Answer»** стає `disabled`.
   - **Авто-відтворення:** Якщо реалізовано TTS, додаток автоматично програє аудіофайл еталона (або генерує звук «на льоту» через OS-модуль мовлення). Користувач може повторно прослухати фразу за допомогою іконки «Play».

## 4. Макет інтерфейсу (UI Layout)

Дизайн має бути мінімалістичним, з високим рівнем контрасту, оптимізованим під швидку роботу «наосліп» з клавіатури.

```
+-------------------------------------------------------------+
|                        AUDIO DRILL                          |
|                     [ Card 3 of 14 ]                        |
+-------------------------------------------------------------+
|                                                             |
|  TASK:                                                      |
|  "Say that the medical professional officially advised      |
|   sending the patient to a hospital."                       |
|                                                             |
+-------------------------------------------------------------+
|                                                             |
|  TARGET (Hidden by default):                                |
|  » recommended hospitalization «   [Icon: Play Audio]       |
|                                                             |
+-------------------------------------------------------------+
|       [ Spacebar: Show Answer ]      [ Enter: Next Drill ]  |
+-------------------------------------------------------------+
```

### Гарячі клавіші (Обов'язково для Fluent-воркфлоу)

- `Space` — Показати відповідь (як в Anki).
- `Enter` / `Right Arrow` — Наступне завдання.
- `R` — Повторити аудіо-еталон.

## 5. Технічні інструкції для реалізації на Python (PySide6)

Для десктопного додатка логіка реалізується через зв'язку елементів керування у `QWidget`.

### Основні компоненти:

- `QLabel` для виведення тексту завдання (`task`) з увімкненим переносом рядків (`setWordWrap(True)`).
- Контейнер `QFrame` для блоку відповіді, стиль якого перемикає видимість (`setVisible(False)` $\rightarrow$ `setVisible(True)`).
- `QMediaPlayer` + `QAudioOutput` для програвання аудіо-файлів з папки рендерингу уроку.

### Сніппет ключової логіки перемикання станів (PySide6/Python):

Python

```
import random
from PySide6.QtWidgets import QWidget, QVBoxLayout, QLabel, QPushButton, QHBoxLayout
from PySide6.QtCore import Qt

class DrillWidget(QWidget):
    def __init__(self, drills_data):
        super().__init__()
        self.drills = drills_data  # Масив об'єктів з JSON
        self.history = []
        self.current_index = -1
        self.init_ui()

    def init_ui(self):
        self.layout = QVBoxLayout(self)
        
        # Елементи інтерфейсу
        self.lbl_task = QLabel("Click 'Next Drill' to start practicing.")
        self.lbl_task.setAlignment(Qt.AlignmentFlag.AlignCenter)
        self.lbl_task.setWordWrap(True)
        
        self.lbl_target = QLabel("")
        self.lbl_target.setAlignment(Qt.AlignmentFlag.AlignCenter)
        self.lbl_target.setStyleSheet("color: #2563eb; font-weight: bold; font-size: 18px;")
        self.lbl_target.hide() # Сховано на старті
        
        # Кнопки керування
        self.btn_show = QPushButton("Show Answer (Space)")
        self.btn_next = QPushButton("Next Drill (Enter)")
        self.btn_show.setEnabled(False)
        
        self.btn_show.clicked.connect(self.show_answer)
        self.btn_next.clicked.connect(self.next_drill)
        
        # Розміщення
        btn_layout = QHBoxLayout()
        btn_layout.addWidget(self.btn_show)
        btn_layout.addWidget(self.btn_next)
        
        self.layout.addWidget(self.lbl_task)
        self.layout.addWidget(self.lbl_target)
        self.layout.addLayout(btn_layout)

    def next_drill(self):
        if not self.drills:
            return
            
        if len(self.history) >= len(self.drills):
            self.history.clear() # Обнулення циклу
            
        available_indices = [i for i in range(len(self.drills)) if i not in self.history and i != self.current_index]
        
        # Крайній випадок, якщо елемент один
        if not available_indices:
            available_indices = [0]
            
        self.current_index = random.choice(available_indices)
        self.history.append(self.current_index)
        
        # Оновлення UI
        self.lbl_task.setText(self.drills[self.current_index]["task"])
        self.lbl_target.setText(self.drills[self.current_index]["target"])
        self.lbl_target.hide()
        
        self.btn_show.setEnabled(True)
        self.btn_next.setEnabled(False) # Змушуємо спочатку подивитися відповідь

    def show_answer(self):
        self.lbl_target.show()
        self.btn_show.setEnabled(False)
        self.btn_next.setEnabled(True)
```

## 6. Критерії готовності (Definition of Done)

1. **Повна ізоляція мови:** У тасках і відповідях немає українських літер. Тільки англійські описи концептів.
2. **Захист від дублів:** Алгоритм випадкового вибору не показує одну й ту саму картку двічі поспіль, доки не пройдено весь пул завдань.
3. **Keyboard-Driven:** Тренажер повністю проходить цикл «Опис $\rightarrow$ Промовляння $\rightarrow$ Перевірка $\rightarrow$ Наступний» за допомогою гарячих клавіш без використання миші.
